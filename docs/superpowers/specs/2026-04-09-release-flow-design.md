# CleanMac Release Flow Design

## Overview

Automate macOS .dmg packaging and GitHub Release creation with changelog, triggered manually via GitHub Actions with version selection.

## Trigger

- **Mechanism**: GitHub Actions workflow dispatch with version selector (major/minor/patch)
- **Version source**: GitHub Release tag (e.g. `v1.0.0`), incremented based on selected version type
- **No auto-trigger**: Never triggers on push; user manually runs from Actions tab

## CI Flow

```
User selects version → Runs workflow → Build → Package DMG → Bump version + Tag → Push → Create Release
```

### Steps

1. **Checkout** — `actions/checkout@v4` with `fetch-depth: 0`
2. **Setup Xcode** — `maxim-lobanov/setup-xcode@v1` with Xcode 15+
3. **Configure Git** — set user email/name for commit
4. **Build** — `xcodebuild` compile CleanMac.app
5. **Package DMG** — `hdiutil create -srcfolder` from built .app
6. **Generate Changelog** — `git-cliff --unreleased --tag <version>` parse Conventional Commits
7. **Bump version** — `git-cliff --bump` increment version, commit and tag
8. **Push** — `git push origin main && git push origin v{version}`
9. **Create Release** — `softprops/action-gh-release@v1` with .dmg asset and changelog

### Build Failure Handling

- If build fails → workflow exits with error, **no tag pushed, no release created, git history unchanged**
- If DMG creation fails → workflow exits with error, **no release**
- If any step fails after build → clean state, no partial artifacts committed

## Artifacts

- `CleanMac-{version}.dmg` — compressed DMG, user download file
- Release body — changelog generated by `git-cliff` from Conventional Commits

## Release Creation

- **Type**: Draft Release (user reviews before publishing) OR direct publish (configurable)
- **Tag**: `v{version}` (e.g. `v1.0.0`), created by workflow
- **Action**: `softprops/action-gh-release@v1` uploads .dmg asset and sets changelog body

## Conventional Commits

Commits must follow [Conventional Commits](https://www.conventionalcommits.org/):

| Type | Description |
|------|-------------|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `docs:` | Documentation |
| `refactor:` | Code change |
| `perf:` | Performance |
| `chore:` | Build/工具 change |

## Key Decisions

- **`hdiutil` over `create-dmg`**: Built into macOS, no external tool dependency in CI
- **`softprops/action-gh-release`**: Standard action for release creation, handles asset upload and body templating
- **`git-cliff --bump`**: Handles version incrementing and tagging in one step
- **Draft Release**: Default to draft so user can review changelog before publishing; can toggle to direct publish via `draft: false`
- **Xcode 15+**: Latest stable, good macOS 13+ support
